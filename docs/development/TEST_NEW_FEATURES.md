# 新功能测试指南

```bash
cd /Users/ztw/Documents/dev/MyFundSys/frontend

```

访问 http://localhost:5173/MyFundSys/

## 测试清单

### 1. 密码登录 ✅

**测试步骤：**
1. 打开页面，显示密码输入界面
2. 输入错误密码（如 123）
3. 点击"进入系统"
4. 验证：显示"密码错误"
5. 输入正确密码：888
6. 验证：进入主界面

**预期结果：** 密码正确才能进入

---

### 2. 离线优先数据操作 ✅

**测试步骤：**
1. 进入系统后，打开浏览器开发者工具
2. 在 Network 面板中，勾选"Offline"
3. 添加一笔交易记录
4. 验证：交易成功添加到本地
5. 取消"Offline"，恢复网络
6. 验证：数据自动同步到云端

**预期结果：** 离线时可正常操作，恢复网络后自动同步

---

### 3. 同步状态显示 ✅

**测试步骤：**
1. 查看顶部状态栏
2. 验证：显示"在线"或"离线"
3. 添加交易记录
4. 验证：如有待同步数据，显示"X 待同步"
5. 点击"立即同步"按钮
6. 验证：同步完成后显示"刚刚"

**预期结果：** 同步状态实时更新

---

### 4. 定时任务配置 ✅

**测试步骤：**
1. 进入"设置"页面
2. 找到"定时任务"卡片
3. 验证：显示3个预设任务（自动抓取净值、生成日报、飞书推送）
4. 点击任务开关，启用/禁用
5. 点击"执行"按钮
6. 验证：显示执行结果（成功/失败）

**预期结果：** 可以配置任务，手动执行测试

---

### 5. 飞书推送配置 ✅

**测试步骤：**
1. 在飞书创建一个群
2. 添加自定义机器人，复制 Webhook URL
3. 进入"设置" → "飞书推送"
4. 点击"编辑配置"
5. 粘贴 Webhook URL
6. 勾选通知类型
7. 点击"保存"
8. 点击"测试连接"
9. 验证：飞书群收到测试消息

**预期结果：** 配置成功后可以发送测试消息

---

### 6. 多端同步测试 ✅

**测试步骤（需要两台设备）：**
1. 设备A：添加一笔交易
2. 设备B：刷新页面
3. 验证：设备B显示新添加的交易
4. 设备B：删除该交易
5. 设备A：等待几秒或手动同步
6. 验证：设备A交易被删除

**预期结果：** 数据在多端自动同步

---

### 7. 数据导入导出 ✅

**测试步骤：**
1. 添加一些交易记录
2. 进入"设置" → "导出数据"
3. 验证：下载 JSON 文件
4. 清空浏览器数据（或换浏览器）
5. 进入系统，输入密码
6. 进入"设置" → "导入数据"
7. 选择刚才导出的文件
8. 验证：数据恢复成功

**预期结果：** 数据可以完整导出和导入

---

## 故障排查

### 问题1：同步失败

**检查：**
1. 查看浏览器控制台错误信息
2. 检查 Supabase 配置是否正确
3. 检查网络连接

### 问题2：定时任务无法执行

**说明：** 这是预期行为。浏览器无法运行真正的定时任务，需要后端支持。

### 问题3：飞书测试失败

**检查：**
1. Webhook URL 是否正确复制
2. 是否包含多余的空格
3. 飞书机器人是否被删除
