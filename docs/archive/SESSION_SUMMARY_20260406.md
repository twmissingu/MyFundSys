## 二、持仓批次管理

### 2.1 从聚合持仓改为批次派生

**Before**: holdings 表按基金聚合，一笔一条记录  
**After**: 从 transactions 派生批次（Lot），每笔买入独立追踪

```
deriveLots(transactions) → Lot[]
  - 买入交易 → 批次（含在途买入）
  - 卖出交易 → 按成本最低匹配扣减批次份额
  - 批次 remainingShares === 0 → 移入已实现盈亏
```

### 2.2 在途买入展示

- pending 买入交易自动出现在持仓明细
- 灰色卡片，显示金额和"净值待定"
- 净值确认后自动变为正常批次

### 2.3 卖出匹配逻辑

- 按成本升序匹配（先卖成本最低的）
- 依次扣减批次剩余份额
- 份额 < 0.01 视为已卖完（浮点数精度处理）

## 五、数据流

```
用户添加买入交易 → transactions 表
    ↓
deriveLots() 派生批次 → 持仓明细
    ↓
用户卖出 → 按成本最低匹配扣减批次
    ↓
批次 remainingShares < 0.01 → 移入落袋为安
    ↓
总资产 = Σ(批次市值) + 在途买入
浮动盈亏 = 总资产 - 总成本
累计盈亏 = 浮动盈亏 + 已实现盈亏
```
