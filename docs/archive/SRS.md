## 1. 引言

### 1.1 项目背景
MyFundSys 是一个基于 E大（ETF拯救世界）投资理念的个人基金投资管理系统。旨在帮助投资者跟踪持仓、管理交易、分析盈亏，辅助投资决策。

### 1.2 目标用户
- 个人基金投资者（单用户）
- 指数ETF投资者

### 1.3 项目目标
- 提供便捷的持仓管理和交易记录功能
- 按批次追踪每笔买入，支持精确盈亏计算
- 实时计算持仓市值和浮动盈亏
- 追踪已实现盈亏（落袋为安）
- 数据存储在 Supabase 云端，支持多设备访问

## 4. 技术架构

### 4.1 前端架构
```
React 18 + TypeScript
├── Vite (构建工具)
├── Ant Design Mobile (UI组件)
├── Supabase Client (后端服务)
└── Recharts (图表库)
```

### 4.2 后端架构
```
Supabase (BaaS)
├── PostgreSQL (数据库)
├── Row Level Security (权限控制)
└── Edge Functions (Deno)
    ├── fund-nav (基金净值)
    ├── fund-search (基金搜索)
    └── fund-history (历史净值)
```

### 4.3 数据模型
- **transactions**: 交易记录（买入/卖出，含在途状态）
- **holdings**: 持仓汇总（从交易派生，仅用于兼容）
- **favorite_funds**: 基金收藏
- **fund_cache**: 搜索缓存

### 4.4 数据派生模型（内存）
- **Lot（批次）**: 从 completed 买入交易派生
- **RealizedLot（已实现）**: 从全部卖出的批次派生
- **HoldingSummary（汇总）**: 从批次按基金聚合

## 6. 核心业务逻辑

### 6.1 批次派生
```
所有 completed 买入交易 → 批次列表
  ↓
按成本升序匹配卖出交易
  ↓
批次 remainingShares > 0 → 持仓明细
批次 remainingShares < 0.01 → 落袋为安
```

### 6.2 卖出匹配
```
用户提交卖出份额
  ↓
查找该基金所有批次（按成本升序）
  ↓
依次扣减，直到卖出份额满足
  ↓
创建卖出交易记录
  ↓
批次 remainingShares < 0.01 → 自动移入落袋为安
```

### 6.3 在途交易处理
```
用户添加买入（非交易日/未来日期）
  ↓
status: 'pending' 写入 transactions
  ↓
页面刷新时 processPendingTransactions() 检查
  ↓
净值已更新 → status → 'completed'
  ↓
前端自动从 transactions 派生新批次
```

### 6.4 总资产计算
```
总资产 = Σ(最新净值 × 批次份额) + 在途买入金额
总成本 = Σ(批次成本) + 在途买入金额
浮动盈亏 = 总资产 - 总成本
累计盈亏 = 浮动盈亏 + 已实现盈亏
```

| 问题 | 影响 | 解决方案 |
|------|------|----------|
| 重复交易记录 | 批次 key 冲突 | 使用 index 作为 key |
