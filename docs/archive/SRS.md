## 1. 引言

### 1.1 项目背景
MyFundSys 是一个基于 E大（ETF拯救世界）投资理念的专业基金投资管理系统。旨在帮助投资者跟踪持仓、管理交易、学习投资理念，并通过数据驱动的策略辅助投资决策。

### 1.2 目标用户
- 个人基金投资者
- E大投资理念学习者
- 指数ETF投资者
- 希望实现资产多元化配置的用户

### 1.3 项目目标
- 提供便捷的持仓管理和交易记录功能
- 实现跨设备数据同步
- 集成 E大投资文章和策略
- 提供基础的数据分析和回测功能

## 3. 非功能需求

### 3.1 性能需求
- 页面加载时间 < 3秒
- 数据查询响应时间 < 1秒
- 支持同时在线用户 > 1000

### 3.2 可用性需求
- 系统可用性 > 99.5%
- 支持离线模式（本地存储）
- 数据自动备份

### 3.3 安全需求
- 用户密码加密存储
- 数据传输 HTTPS
- RLS 行级安全策略
- JWT Token 认证

### 3.4 兼容性需求
- 支持 Chrome/Firefox/Safari/Edge
- 支持移动端浏览器
- 响应式布局设计

## 4. 技术架构

### 4.1 前端架构
```
React 18 + TypeScript
├── Vite (构建工具)
├── Ant Design Mobile (UI组件)
├── Supabase Client (后端服务)
├── Dexie (IndexedDB封装)
└── Recharts (图表库)
```

### 4.2 后端架构
```
Supabase (BaaS)
├── PostgreSQL (数据库)
├── Row Level Security (权限控制)
├── Realtime (实时订阅)
└── Auth (身份认证)
```

### 4.3 数据模型
- **funds**: 基金基础信息
- **holdings**: 用户持仓
- **transactions**: 交易记录
- **strategies**: 投资策略
- **articles**: E大文章

## 6. 待开发功能清单

### 6.1 高优先级（P1）
1. **真实基金净值API接入** - 解决CORS限制，接入天天基金网或其他数据源
2. **真实市场估值数据** - 接入全市场PE/PB数据源
3. **密码重置功能** - 完善用户认证流程
4. **基金详情页** - 展示基金详细信息和历史数据

### 6.2 中优先级（P2）
1. **文章自动抓取** - 从公众号、雪球等源自动同步E大文章
2. **自定义策略创建** - 允许用户创建自己的投资策略
3. **策略信号提醒** - 当策略条件满足时发送提醒
4. **CSV导入/导出** - 支持Excel格式的数据交换
5. **收益日报/周报** - 定期生成投资收益报告

### 6.3 低优先级（P3）
1. **基金收藏/关注** - 关注感兴趣的基金
2. **文章收藏** - 收藏喜欢的文章
3. **持仓分类统计** - 按行业/地区等维度统计
4. **交易记录批量导入** - 支持历史数据批量导入

## 7. 已知问题

| 问题 | 影响 | 解决方案 |
|------|------|----------|
| 基金净值为模拟数据 | 持仓市值不准确 | 接入真实API或后端代理 |
| 市场估值为模拟数据 | 估值判断不准确 | 接入真实估值数据源 |
| 文章为静态示例 | 内容有限 | 开发文章抓取功能 |
| 策略回测为模拟数据 | 回测结果不准确 | 接入历史净值数据 |
