# 基金系统 - 投资建议规则 (E大风格)

## 1. 概述

本文档定义基于E大（ETF拯救世界）投资理念的投资建议规则，包括估值判断、仓位建议和买卖信号三大模块。

E大投资理念核心：
- **低估买入，高估卖出**：基于估值百分位进行逆向投资
- **分散配置**：股债商海外多资产配置
- **长期持有**：不追涨杀跌，耐心持有
- **网格交易**：在低估区间分批买入，高估区间分批卖出

## 2. 估值判断规则

### 2.1 估值等级定义

基于PE/PB百分位综合判断估值水平：

| 估值等级 | 代码 | PE百分位 | PB百分位 | 综合温度 | 颜色标识 |
|----------|------|----------|----------|----------|----------|
| 极度低估 | 1 | < 10% | < 10% | 0-15 | 🔵 深蓝 |
| 低估 | 2 | 10% - 30% | 10% - 30% | 15-35 | 🟢 绿色 |
| 合理偏低 | 3 | 30% - 45% | 30% - 45% | 35-50 | 🟡 浅黄 |
| 合理 | 4 | 45% - 55% | 45% - 55% | 50 | ⚪ 白色 |
| 合理偏高 | 5 | 55% - 70% | 55% - 70% | 50-65 | 🟠 橙色 |
| 高估 | 6 | 70% - 90% | 70% - 90% | 65-85 | 🔴 红色 |
| 极度高估 | 7 | > 90% | > 90% | 85-100 | 🟣 紫色 |

### 2.2 估值计算方法

#### 综合温度计算
```
估值温度 = (PE百分位 × 0.6 + PB百分位 × 0.4) × 100
```

> 注：PE权重更高，因为盈利是长期回报的核心来源

#### 估值等级判定逻辑
```python
def get_valuation_level(pe_percentile, pb_percentile):
    temperature = pe_percentile * 0.6 + pb_percentile * 0.4
    
    if temperature < 0.10:
        return 1  # 极度低估
    elif temperature < 0.30:
        return 2  # 低估
    elif temperature < 0.45:
        return 3  # 合理偏低
    elif temperature < 0.55:
        return 4  # 合理
    elif temperature < 0.70:
        return 5  # 合理偏高
    elif temperature < 0.90:
        return 6  # 高估
    else:
        return 7  # 极度高估
```

## 3. 仓位建议规则

### 3.1 目标仓位矩阵

根据估值等级建议的目标仓位比例：

| 估值等级 | 估值状态 | 建议仓位 | 仓位区间 | 操作方向 |
|----------|----------|----------|----------|----------|
| 1 - 极度低估 | 历史大底 | 90-100% | 重仓 | 积极买入 |
| 2 - 低估 | 便宜区间 | 70-90% | 较重 | 分批买入 |
| 3 - 合理偏低 | 略便宜 | 50-70% | 中等 | 正常配置 |
| 4 - 合理 | 合理价 | 40-50% | 标配 | 持有观望 |
| 5 - 合理偏高 | 略贵 | 20-40% | 较轻 | 谨慎持有 |
| 6 - 高估 | 偏贵区间 | 5-20% | 轻仓 | 分批卖出 |
| 7 - 极度高估 | 历史高位 | 0-5% | 清仓 | 果断卖出 |

### 3.2 仓位计算公式

```
目标仓位 = 基准仓位 × 估值系数

其中：
- 基准仓位：根据风险偏好设定（保守40%/稳健60%/进取80%）
- 估值系数：根据估值等级调整
```

#### 估值系数对照表
| 估值等级 | 估值系数 |
|----------|----------|
| 1 - 极度低估 | 1.5 |
| 2 - 低估 | 1.2 |
| 3 - 合理偏低 | 1.0 |
| 4 - 合理 | 0.8 |
| 5 - 合理偏高 | 0.5 |
| 6 - 高估 | 0.2 |
| 7 - 极度高估 | 0.0 |

### 3.3 仓位调整建议

当实际仓位与目标仓位偏离超过阈值时，给出调整建议：

| 偏离幅度 | 建议操作 |
|----------|----------|
| > 20% | 强烈建议调整 |
| 10% - 20% | 建议逐步调整 |
| 5% - 10% | 可小幅调整 |
| < 5% | 维持现状 |

## 4. 买入信号规则

### 4.1 买入条件矩阵

| 估值等级 | 买入信号 | 买入强度 | 单笔金额建议 | 买入频率 |
|----------|----------|----------|--------------|----------|
| 1 - 极度低估 | 强烈买入 | ★★★★★ | 基准金额 × 2 | 每周可买 |
| 2 - 低估 | 积极买入 | ★★★★☆ | 基准金额 × 1.5 | 每两周可买 |
| 3 - 合理偏低 | 正常买入 | ★★★☆☆ | 基准金额 × 1 | 每月可买 |
| 4 - 合理 | 观望/定投 | ★★☆☆☆ | 基准金额 × 0.5 | 每月定投 |
| 5+ | 暂停买入 | ☆☆☆☆☆ | 0 | 不买 |

### 4.2 买入金额计算

```
单笔买入金额 = 基准定投金额 × 估值系数 × 资金系数

其中：
- 基准定投金额：用户设定的常规定投金额
- 估值系数：根据估值等级（见上表）
- 资金系数：根据可用资金调整（默认1.0）
```

### 4.3 买入策略建议

#### 网格买入法（适用于低估区间）
```
极度低估区间（温度<15）：每下跌5%，加仓一次
低估区间（温度15-35）：每下跌8%，加仓一次
合理偏低（温度35-50）：每下跌10%，加仓一次
```

#### 定投买入法（适用于合理区间）
```
定期定额：每月固定日期投入固定金额
定期不定额：根据估值温度调整金额
  - 温度<30：金额 × 1.5
  - 温度30-50：金额 × 1.0
  - 温度50-70：金额 × 0.5
  - 温度>70：暂停定投
```

## 5. 卖出信号规则

### 5.1 卖出条件矩阵

| 估值等级 | 卖出信号 | 卖出强度 | 建议操作 |
|----------|----------|----------|----------|
| 7 - 极度高估 | 强烈卖出 | ★★★★★ | 清仓或留底仓 |
| 6 - 高估 | 积极卖出 | ★★★★☆ | 分批减仓50%+ |
| 5 - 合理偏高 | 谨慎卖出 | ★★★☆☆ | 适当减仓20% |
| 4 - 合理 | 持有观望 | ★★☆☆☆ | 正常持有 |
| 3- | 坚决持有 | ☆☆☆☆☆ | 不卖 |

### 5.2 分批卖出策略

#### 金字塔卖出法
```
极度高估（温度>90）：每次卖出持仓的30-50%
高估（温度85-90）：每次卖出持仓的20-30%
合理偏高（温度70-85）：每次卖出持仓的10-20%
```

#### 目标收益率卖出法
```
达到目标收益（如年化15%）：可减仓30%
收益翻倍：可减仓50%
收益翻两倍：可减仓70%或清仓
```

### 5.3 止盈止损规则

| 场景 | 建议操作 |
|------|----------|
| 单只基金收益 > 50% | 考虑减仓30% |
| 单只基金收益 > 100% | 考虑减仓50%或清仓 |
| 单只基金亏损 > 20% | 检查基本面，可考虑补仓 |
| 单只基金亏损 > 30% | 评估是否继续持有 |

## 7. 特殊场景处理

### 7.1 新基金建仓

对于无历史估值数据的新基金：
- 参考同类基金估值
- 采用定投方式建仓
- 前3个月不评估估值等级

### 7.2 长期无估值数据

对于无法获取估值数据的基金（如主动基金）：
- 参考业绩比较基准估值
- 结合净值走势判断
- 降低该基金的配置权重

### 7.3 市场极端情况

| 场景 | 处理策略 |
|------|----------|
| 熔断/跌停 | 暂停交易建议，等待市场稳定 |
| 连续大涨 | 提高估值等级，加速卖出 |
| 连续大跌 | 降低估值等级，分批买入 |
| 黑天鹅事件 | 人工介入，暂停自动建议 |

## 8. 规则配置参数

### 8.1 可配置参数

| 参数名 | 默认值 | 说明 |
|--------|--------|------|
| pe_weight | 0.6 | PE权重 |
| pb_weight | 0.4 | PB权重 |
| base_position_conservative | 0.4 | 保守型基准仓位 |
| base_position_moderate | 0.6 | 稳健型基准仓位 |
| base_position_aggressive | 0.8 | 进取型基准仓位 |
| deviation_threshold | 0.05 | 仓位偏离阈值 |
| buy_grid_interval | 0.05 | 网格买入间隔 |
| sell_grid_interval | 0.05 | 网格卖出间隔 |

### 8.2 参数调整影响

```
提高PE权重 → 更看重盈利能力，对成长股更友好
降低偏离阈值 → 更频繁的仓位调整建议
缩小网格间隔 → 更频繁的买卖操作
```
